Submitting a build to the dashboard


Contents: [Setup]   [Unix]   [Windows]  

Setup

  1. Obtain and install Dart from KitWare. Call the installation directory $DART.
  2. Checkout the VXL source tree into $VXLSRC. Checking out as anonymous is fine. Due to Dart implementation details, each build will need it's own source tree.
  3. Create the binary directory $VXLBIN.
  4. Run the initial CMake: 
    cd $VXLBIN && cmake -DDART_ROOT:PATH=$DART $VXLSRC
You will need to have Tcl installed your machine, since Dart is written in Tcl. Make sure the cache $VXLBIN/CMakeCache.txt has an entry for TCL_SH.

Building on Unix

  1. Update Dart via cvs: 
    cd $DART && cvs update -Pd
  2. Clean the binary directory: 
    cd $VXLBIN && make clean
    or, even cleaner, 
    rm -rf $VXLBIN/* && cd $VXLBIN && cmake -DDART_ROOT:PATH=$DART $VXLSRC
  3. Build. 
    tclsh $DART/Source/Client/DashboardManager.tcl DartConfiguration.tcl Nightly Start Update Configure Build Test Submit

Building on Windows

  1. Update Dart.
  2. Build: 
    cd $VXLBIN
tclsh $DART/Source/Client/DashboardManager.tcl DartConfiguration.tcl Nightly Start Update Configure Build Test Submit
